# Name Component Pool Expansion Guide

## Current Problem

**Existing pools are too small:**
- 10-15 prefixes × 10 middles × 10 suffixes = ~1,500 max combinations
- Many combinations unpronounceable
- Will feel repetitive quickly
- Not enough variety for thousands of players

**Solution:** Expand each component pool to 50-100+ options

---

## Target Pool Sizes

**For each archetype/philosophy combination:**
- **Prefixes:** 50-100 options
- **Middles:** 50-100 options  
- **Suffixes:** 30-50 options
- **Total combinations:** 75,000 - 500,000 per pattern
- **Multiple patterns:** 2-4 per archetype = 150,000 - 2,000,000 total

## Systematic Expansion Method

### For Each Archetype:

**Step 1: Analyze phonetic patterns**
- Identify authentic sounds (consonants, vowels, clusters)
- Note cultural phonetic rules (Germanic "sch-", Greek "ph-", etc.)
- Document forbidden combinations (unpronounceable)

**Step 2: Generate prefix matrix**
- Start consonants (20-30 options)
- Consonant + vowel combos (100+ options)
- Consonant clusters + vowel (50+ options)

**Step 3: Generate middle matrix**
- Single consonants (10-20)
- Double consonants (20-40)
- Consonant-vowel-consonant (100+)
- Cultural specific clusters (20-50)

**Step 4: Generate suffix matrix**
- Cultural endings (10-20 base)
- Vowel variations of each (30-50)
- Compound suffixes (10-20)

**Step 5: Validation rules**
```javascript
function isPronounceable(name) {
  // Check for too many consonants in a row
  if (name.match(/[^aeiou]{4,}/)) return false;
  
  // Check for too many vowels in a row
  if (name.match(/[aeiou]{4,}/)) return false;
  
  // Check minimum length
  if (name.length < 4) return false;
  
  // Check maximum length
  if (name.length > 15) return false;
  
  return true;
}
```

## Recommended Approach: Component Database

Instead of hardcoding massive arrays, store components in database/JSON:

### File Structure
```
src/game/nameGeneration/
├── components/
│   ├── engineer.json
│   ├── mountain.json
│   ├── ghost.json
```

### Example: engineer.json
```json

          "Ga", "Ge", "Gi", "Go", "Gu",

          "rc", "ct", "rt", "nt", "lt",

          "us", "ius", "eus", "aeus", "oeus",

```

### Generator loads components:
```javascript
const engineerComponents = require('./components/engineer.json');

generateEngineerGlory() {
  const pattern = this.random(engineerComponents.glory.patterns);
  const prefix = this.random(pattern.prefixes);
  const middle = pattern.middles ? this.random(pattern.middles) : '';
  const suffix = this.random(pattern.suffixes);
  
  const name = prefix + middle + suffix;
  
  return this.isPronounceable(name) ? name : this.generateEngineerGlory();
}
```
